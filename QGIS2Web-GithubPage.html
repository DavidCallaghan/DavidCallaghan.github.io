QGIS to Leaflet to Github Pages

Step One: QGIS

Build your map in QGIS. Get everything looking the way you want it to. Prepare your symbology, the names of your layers and make everything look as you would want it to look in your final product.

  1. Note: everything might have to be geoJSONs, but not sure

Add the plugin “QGIS2Web” to your QGIS instance. Select "Make a map" in the plugin's dropdown menu. Fiddle around with the options. Press "Export". The folder structure and files that produce your webpage will be exported to the folder path you selected in the previous options.

Step Two: Github Pages

Go to https://pages.github.com/, sign in to your Github account and follow the instructions to create a “Repository” (aka "Repo", aka the bit of Github where you are going to host your web map) on Github online. Download Github Desktop from the same page. Once Github Desktop has downloaded, return to your Github online account, navigate to the new Repo and add (aka "clone") it to your Github Desktop by clicking the button that says “Setup in desktop”. This will download a copy of that new folder onto your local machine.

Step Three: Your PC or Mac’s file explorer

Once the Repo folder is on your local machine, you can copy the complete folder structure and HTML file for the webmap into that new Repo folder. Make sure that you copy all the content from the QGIS2Web folder but not the folder itself.

Step Four: Github Desktop

Github Desktop should detect that you’ve made changes to the repository. Bottom left click “Commit” (you may need to write a comment in the box above the button). It’ll do some thinking, then take you to another page in the app that has a button that says “Push”, click that and wait till it does its stuff.

Step Five: Access webmap online

Open your browser and navigate to https://your-github-account-name.github.io/filename-of-html-page-exported-from-qgis.html and you should see your map.
